# Benchmark Workstream Batch

You are part of a batch of agents implementing benchmarking infrastructure. All benchmark source lives under `src/rietveld_next/benchmarks/` or the existing benchmark package under `src/`. Benchmarks must be deterministic, large benchmarks opt-in, normal CI smoke-only, and every benchmark emits machine-readable JSON. Benchmark output must identify backend, dtype, device/CPU details, size, iterations, median/min/max runtime, checksum, compile time where applicable, and git commit if available. Only one agent may edit the central benchmark schema or main benchmark runner at a time.
