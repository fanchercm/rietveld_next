# Code Of Conduct

## Purpose

Rietveld Next contributors should maintain a respectful, evidence-oriented
environment for scientific software work.

## Scope

This policy applies to repository issues, reviews, docs, code, discussions, and
agent-generated contributions.

## Non-Goals

This document does not replace institutional policies or legal obligations.

## Expectations

- Discuss technical claims with evidence and reproducible commands.
- Treat failed tests and limitations as useful scientific information.
- Avoid harassment, personal attacks, discriminatory language, and bad-faith
  disruption.
- Disclose uncertainty when validation is incomplete.

## Reporting

Report conduct concerns to the project maintainers named by the repository
owner. Maintainers should respond proportionally and document actions when
appropriate.
