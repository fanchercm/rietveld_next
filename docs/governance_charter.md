# Governance Charter

## Purpose

This charter defines lightweight governance for early Rietveld Next development.

## Scope

The repository currently uses maintainer-led review. Scientific behavior,
schemas, public APIs, and release artifacts require explicit review evidence.

## Non-Goals

This charter does not establish a formal foundation, steering committee, or
voting process.

## Decision Rules

- Scientific formulas require tests, references or fixtures, and docs.
- Persistent file formats require versioned schemas and migration notes.
- Expensive benchmarks and facility workflows must stay opt-in.
- AI-generated code must be reviewed, tested, and validated.
